# 数据库设计

## 数据库信息

- 数据库名：`herb_platform`
- 字符集：`utf8mb4`
- 导入顺序：先 `schema.sql`，后 `data.sql`
- 当前主表数量：23 张
- 统计视图：3 个

## 表结构总览

### 1. 用户权限
- `sys_user` 用户表
- `sys_role` 角色表
- `sys_permission` 权限表
- `sys_user_role` 用户角色关联表
- `sys_role_permission` 角色权限关联表

### 2. 种植管理
- `herb_field` 地块表
- `herb_crop` 作物表
- `herb_farm_record` 农事记录表

### 3. 销售与订单
- `herb_supply` 供应信息表
- `herb_demand` 需求信息表
- `herb_order` 订单表
- `herb_favorite` 收藏表

### 4. 质量溯源
- `herb_trace` 溯源信息表
- `herb_trace_node` 溯源节点表

### 5. 系统辅助
- `sys_config` 系统配置表
- `sys_log` 操作日志表
- `sys_dict` 数据字典表
- `sys_notice` 消息通知表
- `sys_file` 文件记录表
- `stat_daily` 每日统计表

### 6. 种植知识增强
- `herb_knowledge` 中药材百科表
- `herb_disease_case` 病虫害样例表
- `herb_calendar_stage` 种植日历模板表

## 本次关键结构增强

### 1. 四角色体系

`sys_user.user_type` 当前约定：
- `1` 种植户
- `2` 商家
- `3` 管理员
- `4` 普通用户

`data.sql` 已提供四类默认账号和角色映射。

### 2. 销售模块增强

`herb_supply` 新增或重点使用的字段：
- `wholesale_price`：批发价
- `wholesale_min_quantity`：起批量
- `production_area`：产地
- `harvest_date`：采收日期

`herb_order` 已支持：
- 订单状态流转
- 支付状态
- 发货、完成、取消时间
- 物流公司与物流单号

### 3. 溯源模块

`herb_trace` 提供：
- `trace_code`
- `batch_no`
- `production_area`
- `plant_date`
- `harvest_date`
- `quality_standard`

`herb_trace_node` 用于记录从种植到销售的过程节点。

### 4. 知识模块

`herb_knowledge` 用于支撑百科检索，核心字段包括：
- `herb_name`
- `herb_category`
- `planting_season`
- `disease_type`
- `keyword_tags`

`herb_disease_case` 用于病虫害识别演示，核心字段包括：
- `disease_name`
- `symptom_keywords`
- `prevention_plan`
- `image_url`
- `feature_tag`

`herb_calendar_stage` 用于种植日历与提醒，核心字段包括：
- `stage_name`
- `action_type`
- `day_offset`
- `duration_days`
- `reminder_days`
- `operation_window`

## 视图

- `v_user_info`：用户与角色信息汇总
- `v_crop_stats`：作物统计
- `v_order_stats`：订单统计

## 设计说明

- 多数业务表带 `deleted` 字段，采用逻辑删除。
- 多数业务表带 `create_time`、`update_time` 字段，便于审计和排序。
- 供应、订单、溯源、知识模块之间通过主键或业务字段形成逻辑关联。
- 详细字段、索引、外键关系以 `database/schema.sql` 为准。
