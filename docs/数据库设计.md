# 数据库设计

## 数据库信息

- 数据库名: `herb_platform`
- 字符集: `utf8mb4`
- 逻辑删除: 多数业务表包含 `deleted` 字段
- 时间字段: 通常包含 `create_time` / `update_time`

## 表结构总览（20张）

### 用户权限
- `sys_user` 用户表
- `sys_role` 角色表
- `sys_permission` 权限表
- `sys_user_role` 用户-角色关联表
- `sys_role_permission` 角色-权限关联表

### 种植管理
- `herb_field` 地块表
- `herb_crop` 作物表
- `herb_farm_record` 农事记录表

### 销售对接
- `herb_supply` 供应信息表
- `herb_demand` 需求信息表
- `herb_order` 订单表
- `herb_favorite` 收藏表

### 质量溯源
- `herb_trace` 溯源信息表
- `herb_trace_node` 溯源节点表

### 系统辅助
- `sys_config` 系统配置表
- `sys_log` 操作日志表
- `sys_dict` 数据字典表
- `sys_notice` 消息通知表
- `sys_file` 文件记录表
- `stat_daily` 每日统计表

## 详细结构

- 完整字段与索引请查看 `database/schema.sql`。
- 初始化数据请查看 `database/data.sql`。

## 设计说明

- 用户类型: `1-种植户`、`2-采购商`、`3-管理员`。
- 状态字段: 通常 `1-正常`、`0-禁用` 或 `1-有效`、`0-无效`。
- 业务模块使用外键或逻辑关联保证数据一致性。
