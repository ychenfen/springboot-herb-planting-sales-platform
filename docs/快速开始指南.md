# 中药材种植与销售服务平台 - 快速开始指南

## 1. 最快方式：Windows 一键启动

如果你只是想把项目跑起来，直接在仓库根目录执行：

```bat
start.bat
```

脚本默认假设你的电脑上没有 JDK、Node、MySQL、Maven，也不要求预装 Redis。首次运行会自动下载并初始化运行环境。

### 首次运行会做什么

- 下载便携式 JDK 11
- 下载便携式 Node.js LTS
- 下载便携式 MySQL 8.4
- 需要时安装 VC++ Runtime
- 初始化本地数据库 `herb_platform`
- 导入完整表结构和种子数据
- 安装前端依赖并打包后端
- 打开前后端服务窗口

### 默认地址

- 前端：`http://localhost:5173`
- 后端：`http://localhost:8080/api`
- 接口文档：`http://localhost:8080/api/doc.html`

### 本地数据库参数

- Host：`127.0.0.1`
- Port：`13306`
- Database：`herb_platform`
- Username：`herb_local`
- Password：`herb123`

### 默认账号

| 角色 | 用户名 | 密码 |
|---|---|---|
| 管理员 | `admin` | `admin123` |
| 种植户 | `farmer001` | `admin123` |
| 商家 | `buyer001` | `admin123` |
| 普通用户 | `user001` | `admin123` |

## 2. 仅初始化，不立即启动

如果你只想先把环境下好，再稍后启动：

```bat
init-windows.bat
```

该命令会准备 `.runtime/` 下的 JDK、Node、MySQL 和数据库，但不会额外弹出前后端运行窗口。

## 3. 手动开发启动

### 3.1 后端

```bat
cd backend
mvnw.cmd clean package -DskipTests
java -jar target\herb-platform-1.0.0.jar
```

如果要连接一键脚本生成的本地数据库，先设置：

```bat
set DB_HOST=127.0.0.1
set DB_PORT=13306
set DB_NAME=herb_platform
set DB_USERNAME=herb_local
set DB_PASSWORD=herb123
```

说明：
- 开发配置仍保留 `spring.redis.*` 节点，但当前本地演示登录链路不依赖 Redis。
- 文件上传路径默认是 `./uploads`，Windows 下可直接使用。

### 3.2 前端

```bat
cd frontend
npm install
npm run dev
```

如需修改前端代理或端口，可在 `frontend/.env.local` 中调整：

```env
VITE_API_BASE=/api
VITE_PROXY_TARGET=http://localhost:8080
VITE_DEV_PORT=5173
```

## 4. 数据库初始化

### 4.1 全新初始化

```sql
CREATE DATABASE IF NOT EXISTS herb_platform
DEFAULT CHARACTER SET utf8mb4
COLLATE utf8mb4_unicode_ci;

USE herb_platform;
SOURCE database/schema.sql;
SOURCE database/data.sql;
```

### 4.2 旧库升级

```sql
SOURCE database/update_20260311_feature_upgrade.sql;
```

### 4.3 初始化后可验证

```sql
USE herb_platform;
SELECT COUNT(*) FROM sys_user;
SELECT COUNT(*) FROM herb_knowledge;
SELECT COUNT(*) FROM herb_calendar_stage;
```

## 5. 启动后先检查什么

1. 访问 `http://localhost:5173`，确认登录页能打开。
2. 使用 `admin / admin123` 登录，确认能看到完整后台菜单。
3. 打开 `http://localhost:8080/api/doc.html`，确认接口文档正常。
4. 测试知识百科搜索：输入 `黄芪 病害`。
5. 测试病害识别演示：上传任意演示图片。
6. 测试供应下单、智能计价、订单状态流转、溯源信息展示。

## 6. 常见问题

### 6.1 第一次运行很慢

正常。脚本需要下载和解压 JDK、Node、MySQL，并完成一次后端打包和前端依赖安装。

### 6.2 VC++ Runtime 提示或安装失败

便携式 MySQL 在部分 Windows 机器上依赖 VC++ Runtime。脚本会自动尝试安装；若被系统策略拦截，请手动安装后重试。

### 6.3 端口占用

- 前端默认 `5173`
- 后端默认 `8080`
- 本地 MySQL 默认 `13306`

若端口被占用，先关闭对应进程，或按实际情况修改配置后再启动。

### 6.4 想重置本地一键环境

删除仓库下的 `.runtime/` 后重新执行 `init-windows.bat` 或 `start.bat`。
