## 三、后端启动

### 3.1 配置修改

编辑 `backend/src/main/resources/application-dev.yml`（支持环境变量覆盖）:

```yaml
spring:
  datasource:
    url: jdbc:mysql://${DB_HOST:localhost}:${DB_PORT:3306}/${DB_NAME:herb_platform}?...
    username: ${DB_USERNAME:root}
    password: ${DB_PASSWORD:}

  redis:
    host: ${REDIS_HOST:localhost}
    port: ${REDIS_PORT:6379}
    password: ${REDIS_PASSWORD:}
```

文件上传路径配置（`backend/src/main/resources/application.yml`）:

```yaml
herb:
  file:
    access-path: ${HERB_ACCESS_PATH:/uploads}   # 访问路径
    storage-path: ${HERB_STORAGE_PATH:./uploads} # 实际存储路径
```

常用环境变量（Windows/macOS/Linux 通用）:

```
DB_HOST=localhost
DB_PORT=3306
DB_NAME=herb_platform
DB_USERNAME=root
DB_PASSWORD=your_password

REDIS_HOST=localhost
REDIS_PORT=6379
REDIS_PASSWORD=

HERB_ACCESS_PATH=/uploads
HERB_STORAGE_PATH=./uploads
```

### 3.2 方式一：使用IDEA启动

1. 打开 IntelliJ IDEA
2. 选择 File -> Open，打开 `backend` 目录
3. 等待Maven依赖下载完成
4. 找到主类 `HerbPlatformApplication.java`
5. 右键 -> Run 'HerbPlatformApplication'

### 3.3 方式二：使用Maven命令启动

```bash
# 进入后端目录
cd backend

# 清理并安装依赖
mvn clean install

# 启动应用
mvn spring-boot:run
```

### 3.4 方式三：使用jar包启动

```bash
# 打包
cd backend
mvn clean package -Dmaven.test.skip=true

# 运行
java -jar target/herb-platform-1.0.0.jar
```

### 3.5 验证后端启动

访问以下地址验证：
- API文档: http://localhost:8080/api/doc.html
- Druid监控: http://localhost:8080/api/druid (用户名:admin 密码:admin123)

## 七、常见问题

### 7.1 后端启动失败

**问题**: 数据库连接失败
```
Could not open JDBC Connection
```
**解决**:
- 检查MySQL是否启动
- 检查数据库用户名密码是否正确
- 检查数据库 `herb_platform` 是否存在

---

**问题**: Redis连接失败
```
Unable to connect to Redis
```
**解决**:
- 检查Redis是否启动: `redis-cli ping`
- 检查Redis配置（host、port、password）
- 关闭防火墙或添加端口白名单

---

**问题**: 端口被占用
```
Port 8080 was already in use
```
**解决**:
```bash
# Windows - 查找并结束占用进程
netstat -ano | findstr :8080
taskkill /PID <进程ID> /F

# macOS/Linux
lsof -i :8080
kill -9 <PID>

# 或修改application.yml中的端口
server:
  port: 8081
```

---

### 7.2 前端启动失败

**问题**: 依赖安装失败
```
npm ERR! network
```
**解决**:
```bash
# 切换npm源为淘宝镜像
npm config set registry https://registry.npmmirror.com

# 清除缓存重试
npm cache clean --force
npm install
```

---

**问题**: 端口被占用
```
Port 5173 is already in use
```
**解决**:
- 修改 `vite.config.js` 中的端口配置
- 或关闭占用端口的进程

---

### 7.3 跨域问题

如果前后端分离遇到跨域问题，后端已配置CORS，如果仍有问题：

检查后端 `CorsConfig.java` 配置:
```java
.allowedOrigins("http://localhost:5173")
```

## 附录：便捷启动脚本

### Windows (start.bat)
```batch
@echo off
echo Starting Herb Platform...

echo [1/3] Starting Redis...
start redis-server.exe

echo [2/3] Starting Backend...
start cmd /k "cd backend && mvn spring-boot:run"

timeout /t 30

echo [3/3] Starting Frontend...
start cmd /k "cd frontend && npm run dev"

echo All services started!
pause
```

### macOS/Linux (start.sh)
```bash
#!/bin/bash

echo "Starting Herb Platform..."

# 启动Redis
echo "[1/3] Starting Redis..."
brew services start redis

# 启动后端
echo "[2/3] Starting Backend..."
cd backend
mvn spring-boot:run &
BACKEND_PID=$!

# 等待后端启动
sleep 30

# 启动前端
echo "[3/3] Starting Frontend..."
cd ../frontend
npm run dev &
FRONTEND_PID=$!

echo "All services started!"
echo "Backend PID: $BACKEND_PID"
echo "Frontend PID: $FRONTEND_PID"
echo "Press Ctrl+C to stop all services"

wait
```

保存后给脚本执行权限:
```bash
chmod +x start.sh
./start.sh
```

## Windows 配置要点（避免前后端/数据库信息错乱）

- 后端数据库连接只在 `backend/src/main/resources/application-*.yml` 与环境变量中维护。
- 前端不存放数据库信息，仅通过 `/api` 调用后端。
- 若修改后端端口或 `server.servlet.context-path`，务必同步更新 `frontend/vite.config.js` 与 `frontend/.env.local`。
- 上传目录建议使用相对路径（`./uploads`），避免 Windows 路径分隔符问题。
