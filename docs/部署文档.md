# 部署文档

## 环境准备

- JDK 11+
- MySQL 8.0+
- Redis 6.0+
- Node.js 16+ (仅用于前端构建)
- Nginx (推荐用于前端静态资源与反向代理)

## 数据库初始化

```bash
mysql -u root -p < database/schema.sql
mysql -u root -p < database/data.sql
```

## 后端部署

1. 修改配置 `backend/src/main/resources/application-prod.yml`：
   - 数据库连接信息（支持环境变量）
   - Redis连接信息（支持环境变量）
   - 端口与上传目录（`herb.file.access-path` / `herb.file.storage-path`）
   - 日志路径（`LOG_PATH`，Windows 需指定可写路径）

2. 打包并启动：

```bash
cd backend
mvn clean package -DskipTests
java -jar target/*.jar --spring.profiles.active=prod
```

3. 端口与访问：
   - 接口前缀: `/api`
   - API文档: `http://<server>:8080/api/doc.html`

### 环境变量示例

```
DB_USERNAME=herb_user
DB_PASSWORD=your_password
REDIS_HOST=localhost
REDIS_PORT=6379
REDIS_PASSWORD=
HERB_ACCESS_PATH=/uploads
HERB_STORAGE_PATH=./uploads
LOG_PATH=/var/logs/herb-platform/app.log
```

### 可选：systemd 服务

将启动命令写入 systemd service，保证服务开机自启与自动重启。

## 前端部署

1. 构建静态文件：

```bash
cd frontend
npm install
npm run build
```

2. 将 `frontend/dist` 部署到 Nginx 目录，例如：`/var/www/herb-platform`。

## Nginx 配置示例

```nginx
server {
  listen 80;
  server_name your-domain.com;

  root /var/www/herb-platform;
  index index.html;

  location / {
    try_files $uri $uri/ /index.html;
  }

  location /api/ {
    proxy_pass http://127.0.0.1:8080/api/;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
  }
}
```

## Windows 部署要点

- `LOG_PATH` 必须设置为 Windows 可写路径，例如 `C:/logs/herb-platform/app.log`。
- 上传目录建议使用相对路径 `./uploads`，避免路径分隔符问题。
- 若修改端口或 `server.servlet.context-path`，需同步更新前端代理配置。

## 常见问题

- **跨域问题**: 前端走 Nginx 同域代理 `/api`，避免跨域。
- **上传目录权限**: 确保 `uploads` 目录可写（Windows 建议写在用户目录或应用目录内）。
- **端口冲突**: 修改 `application-prod.yml` 或 Nginx 端口配置。
