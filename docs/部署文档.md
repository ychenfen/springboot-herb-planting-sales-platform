# 部署文档

## 1. Windows 本地演示部署

### 1.1 适用场景

用于答辩、演示、快速验收。默认假设本机没有 JDK、Node、MySQL、Maven，也不要求先安装 Redis。

### 1.2 启动方式

在仓库根目录执行：

```bat
start.bat
```

如果只想预下载环境和初始化数据库：

```bat
init-windows.bat
```

### 1.3 首次运行自动完成的内容

- 下载便携式 JDK 11
- 下载便携式 Node.js LTS
- 下载便携式 MySQL 8.4
- 需要时安装 VC++ Runtime
- 初始化本地数据库并导入 `schema.sql` / `data.sql`
- 安装前端依赖
- 使用 Maven Wrapper 打包后端
- 打开前后端运行窗口

### 1.4 本地默认参数

- 前端：`http://localhost:5173`
- 后端：`http://localhost:8080/api`
- 接口文档：`http://localhost:8080/api/doc.html`
- MySQL：`127.0.0.1:13306`
- 数据库：`herb_platform`
- 账号：`herb_local`
- 密码：`herb123`

说明：
- 所有运行时文件都写入仓库下的 `.runtime/`。
- 本地演示版登录态使用内存 token 存储，默认不依赖 Redis。
- 第一次启动下载体积较大，建议保持网络稳定。

## 2. 手动部署后端

### 2.1 环境要求

- JDK 11+
- MySQL 8.0+
- Maven 3.9+ 或直接使用仓库内 `backend/mvnw.cmd`

### 2.2 数据库初始化

```sql
CREATE DATABASE IF NOT EXISTS herb_platform
DEFAULT CHARACTER SET utf8mb4
COLLATE utf8mb4_unicode_ci;

USE herb_platform;
SOURCE database/schema.sql;
SOURCE database/data.sql;
```

旧库升级使用：

```sql
SOURCE database/update_20260311_feature_upgrade.sql;
```

### 2.3 生产配置

修改 `backend/src/main/resources/application-prod.yml` 或通过环境变量覆盖：
- `DB_USERNAME`
- `DB_PASSWORD`
- `SERVER_PORT`
- `HERB_ACCESS_PATH`
- `HERB_STORAGE_PATH`
- `LOG_PATH`

如果你后续需要启用 Redis 缓存，可继续设置：
- `REDIS_HOST`
- `REDIS_PORT`
- `REDIS_PASSWORD`

### 2.4 打包与启动

```bat
cd backend
mvnw.cmd clean package -DskipTests
java -jar target\herb-platform-1.0.0.jar --spring.profiles.active=prod
```

## 3. 手动部署前端

```bat
cd frontend
npm install
npm run build
```

将 `frontend/dist` 发布到 Nginx、IIS 或任意静态站点目录。

## 4. 反向代理示例

```nginx
server {
  listen 80;
  server_name your-domain.com;

  root /var/www/herb-platform;
  index index.html;

  location / {
    try_files $uri $uri/ /index.html;
  }

  location /api/ {
    proxy_pass http://127.0.0.1:8080/api/;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
  }
}
```

## 5. Windows 部署要点

- `LOG_PATH` 必须指向 Windows 可写路径，例如 `C:/logs/herb-platform/app.log`。
- 上传目录建议使用 `./uploads` 或磁盘下明确的应用数据目录。
- 若修改 `server.port` 或 `server.servlet.context-path`，需同步更新前端代理配置。
- 如需彻底重建一键启动生成的本地数据库，删除 `.runtime/data/mysql` 后重新执行 `init-windows.bat`。
