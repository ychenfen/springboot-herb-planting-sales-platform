# 开发指南

## 目录说明

- `backend/` 后端 Spring Boot 项目
- `frontend/` 前端 Vue3 项目
- `database/` 数据库脚本
- `docs/` 项目文档

## 环境准备

- JDK 11+
- Maven 3.6+
- MySQL 8.0+
- Redis 6.0+
- Node.js 16+

## 启动流程

### 后端

```bash
cd backend
mvn spring-boot:run
```

### 前端

```bash
cd frontend
npm install
npm run dev
```

## 开发约定

- 接口统一返回 `Result`，字段包含 `code/message/data/timestamp`。
- 新增接口需同步更新 `docs/API接口文档.md`。
- 统一使用 DTO 接收参数，VO 返回数据。
- 认证采用 JWT，前端通过 `Authorization: Bearer <token>` 传递。

## 前端请求约定

- 请求封装在 `frontend/src/utils/request.js`。
- 默认代理 `/api` 到 `http://localhost:8080`，可通过 `VITE_PROXY_TARGET` 覆盖。
- 401xx 响应将触发重新登录流程。

## 配置一致性约定

- 数据库配置以 `backend/src/main/resources/application-*.yml` 与环境变量为准。
- 上传路径分为访问路径与存储路径：
  - `herb.file.access-path` 用于对外访问（默认 `/uploads`）。
  - `herb.file.storage-path` 为实际存储目录（默认 `./uploads`）。
- 若修改后端端口或 `server.servlet.context-path`，需同步调整前端代理与 `VITE_API_BASE`。

## 权限控制（前端）

- 通过 `GET /system/permission/user` 获取当前用户权限编码。
- 使用指令 `v-permission="'permission:code'"` 控制按钮显示。
- 指令实现位置：`frontend/src/directives/permission.js`。

## 管理员/用户端权限（后端）

- 系统管理相关接口（`/system/*`）为管理员专属，后端已强制校验用户类型。
- 普通用户接口仍按业务模块划分（种植、销售、溯源、通知等）。
- 前端菜单已按 `userType` 与权限码显示，但后端仍以校验为准，防止越权访问。
- 统计分析接口按用户类型过滤数据：管理员看到全局，种植户/采购商看到自身相关统计。
- 业务模块用户类型限制（后端强制校验）:
  - 种植管理（地块/作物/农事记录/溯源管理）仅种植户可操作。
  - 供应信息发布/维护仅种植户可操作，供应大厅对所有登录用户开放。
  - 需求信息发布/维护仅采购商可操作，需求大厅对所有登录用户开放。
  - 订单创建/确认收货仅采购商，发货确认仅种植户，取消订单允许种植户/采购商。

## 系统日志

- 系统日志通过 AOP 记录写操作（POST/PUT/DELETE）。
- 日志入口：`/system/log/page`，可按用户名、操作、时间范围筛选。

## 系统通知

- 管理端使用 `/system/notice` 进行通知管理。
- 用户端通过 `/notice` 查看与标记已读。
