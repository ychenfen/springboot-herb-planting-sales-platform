# 代码生成指南 - 使用Claude Code快速开发

本指南帮助您使用Claude Code快速生成项目所需的各类代码文件。

---

## 一、实体类生成 (Entity)

### 使用Claude Code生成示例

**提示词**：
```
请为用户表(sys_user)生成实体类，包含以下字段：
- id: BIGINT主键
- username: VARCHAR(50)用户名
- password: VARCHAR(100)密码
- realName: VARCHAR(50)真实姓名
- phone: VARCHAR(20)手机号
- email: VARCHAR(100)邮箱
- avatar: VARCHAR(255)头像
- userType: TINYINT用户类型
- status: TINYINT状态
- lastLoginTime: DATETIME最后登录时间
- createTime: DATETIME创建时间
- updateTime: DATETIME更新时间
- deleted: TINYINT逻辑删除

使用Lombok注解，继承MyBatis Plus的BaseEntity
生成到: backend/src/main/java/com/herb/platform/entity/User.java
```

### 批量生成所有实体类

**提示词**：
```
根据database/schema.sql中的表结构，生成以下实体类：
1. User (sys_user)
2. Role (sys_role)
3. Permission (sys_permission)
4. UserRole (sys_user_role)
5. RolePermission (sys_role_permission)
6. Field (herb_field)
7. Crop (herb_crop)
8. FarmRecord (herb_farm_record)
9. Supply (herb_supply)
10. Demand (herb_demand)
11. Order (herb_order)
12. Trace (herb_trace)
13. TraceNode (herb_trace_node)

所有实体类都要：
- 使用Lombok的@Data、@Builder、@NoArgsConstructor、@AllArgsConstructor
- 使用@TableName注解指定表名
- 使用@TableId注解标注主键
- 使用@TableField注解处理特殊字段
- 使用@TableLogic注解标注逻辑删除字段
- 包含完整的字段注释

生成到: backend/src/main/java/com/herb/platform/entity/
```

## 四、Controller层生成

### 单个Controller生成

**提示词**：
```
生成UserController，包含以下接口：

1. GET /users - 分页查询用户列表
2. GET /users/{id} - 获取用户详情
3. POST /users - 创建用户
4. PUT /users/{id} - 更新用户
5. DELETE /users/{id} - 删除用户
6. PUT /users/{id}/status - 更新用户状态
7. POST /users/{id}/roles - 分配角色

要求：
- 使用@RestController和@RequestMapping注解
- 使用@Api和@ApiOperation注解(Swagger)
- 参数校验使用@Valid
- 统一返回Result对象
- 包含权限注解@RequiresPermission

生成到: backend/src/main/java/com/herb/platform/controller/UserController.java
```

### 批量生成Controller

**提示词**：
```
生成以下Controller：

1. AuthController - 认证控制器
   POST /auth/login - 登录
   POST /auth/logout - 登出
   POST /auth/register - 注册
   GET /auth/info - 获取当前用户信息

2. FieldController - 地块控制器
   GET /fields - 查询列表
   GET /fields/{id} - 获取详情
   POST /fields - 添加地块
   PUT /fields/{id} - 更新地块
   DELETE /fields/{id} - 删除地块

3. CropController - 作物控制器
   GET /crops - 查询列表
   GET /crops/{id} - 获取详情
   POST /crops - 添加作物
   PUT /crops/{id} - 更新作物
   PUT /crops/{id}/stage - 更新生长阶段
   POST /crops/{id}/harvest - 记录收获

4. SupplyController - 供应控制器
   GET /supplies - 查询列表
   GET /supplies/search - 搜索供应
   GET /supplies/{id} - 获取详情
   POST /supplies - 发布供应
   PUT /supplies/{id} - 更新供应
   PUT /supplies/{id}/offline - 下架供应

5. OrderController - 订单控制器
   GET /orders - 查询订单列表
   GET /orders/{id} - 获取订单详情
   POST /orders - 创建订单
   PUT /orders/{id}/confirm - 确认订单
   PUT /orders/{id}/cancel - 取消订单

所有Controller都要：
- 添加完整的Swagger注解
- 使用统一Result返回
- 添加参数校验
- 添加权限控制注解

生成到: backend/src/main/java/com/herb/platform/controller/
```

## 五、DTO和VO类生成

### DTO (Data Transfer Object)

**提示词**：
```
为用户模块生成以下DTO类：

1. UserLoginDTO - 登录请求
   - username: 用户名
   - password: 密码

2. UserRegisterDTO - 注册请求
   - username: 用户名
   - password: 密码
   - realName: 真实姓名
   - phone: 手机号
   - email: 邮箱
   - userType: 用户类型

3. UserUpdateDTO - 更新用户
   - realName: 真实姓名
   - phone: 手机号
   - email: 邮箱
   - avatar: 头像

4. PasswordUpdateDTO - 修改密码
   - oldPassword: 旧密码
   - newPassword: 新密码

所有DTO都要：
- 使用Lombok注解
- 添加参数校验注解(@NotBlank、@Email等)
- 添加字段注释和Swagger注解

生成到: backend/src/main/java/com/herb/platform/dto/
```

### VO (View Object)

**提示词**：
```
为用户模块生成以下VO类：

1. UserVO - 用户信息展示
   - id
   - username
   - realName
   - phone
   - email
   - avatar
   - userType
   - status
   - roles: List<RoleVO>
   - createTime

2. UserLoginVO - 登录返回
   - token
   - userInfo: UserVO
   - permissions: List<String>

3. UserPageVO - 用户分页列表项
   - id
   - username
   - realName
   - phone
   - userType
   - status
   - createTime

所有VO都要：
- 使用Lombok注解
- 添加Swagger注解描述字段
- 不包含敏感信息(如密码)

生成到: backend/src/main/java/com/herb/platform/vo/
```

## 八、统一返回和异常处理

### 统一返回类

**提示词**：
```
生成统一返回结果类 Result<T>

包含：
- code: 状态码
- message: 提示信息
- data: 数据
- timestamp: 时间戳

静态方法：
- success(): 成功无数据
- success(T data): 成功带数据
- success(String message, T data): 自定义消息成功
- error(String message): 失败
- error(int code, String message): 自定义状态码失败

生成到: backend/src/main/java/com/herb/platform/common/Result.java
```

## 总结

使用Claude Code生成代码时的最佳实践：

1. **明确需求**: 清晰描述要生成什么，包含哪些功能
2. **提供上下文**: 指明表结构、相关实体类、业务规则
3. **指定规范**: 注解、命名、返回格式等要求
4. **逐层生成**: 先Entity -> Mapper -> Service -> Controller
5. **及时测试**: 生成后立即测试，发现问题及时修正
6. **代码review**: 检查生成的代码是否符合项目规范

有了Claude Code，您可以：
- 快速搭建项目框架
- 批量生成重复性代码
- 提高开发效率
- 专注于业务逻辑

开始使用上述提示词，让Claude Code帮您快速完成项目开发！
