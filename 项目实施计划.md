# 中药材种植与销售服务平台 - 详细实施计划

## 一、项目概述

### 1.1 项目背景
- 中医药产业振兴的国家战略机遇
- 传统农业数字化转型需求
- 解决种植端与销售端信息孤岛问题
- 建立完善的质量溯源体系

### 1.2 项目目标
- 实现种植全流程数字化管理
- 搭建供需对接平台
- 建立质量溯源体系
- 提供数据分析决策支持

### 1.3 预期成果
- 完整的中药材服务平台系统
- 前后端分离的现代化架构
- 可复用的技术方案
- 完善的项目文档

## 二、技术架构设计

### 2.1 整体架构

```
┌─────────────────────────────────────────────┐
│            展现层 (Presentation)             │
│     Vue3 + Element Plus + ECharts           │
└─────────────────┬───────────────────────────┘
                  │ HTTP/HTTPS
┌─────────────────▼───────────────────────────┐
│            应用层 (Application)              │
│        Spring Boot + JWT + Redis            │
│  ┌──────────┬──────────┬──────────┬──────┐ │
│  │ 认证授权  │ 种植管理  │ 销售对接 │ 溯源 │ │
│  └──────────┴──────────┴──────────┴──────┘ │
└─────────────────┬───────────────────────────┘
                  │ MyBatis-Plus
┌─────────────────▼───────────────────────────┐
│            数据层 (Data)                     │
│     MySQL 8.0 + Redis 6.x                   │
└─────────────────────────────────────────────┘
```

### 2.2 技术选型依据

| 技术 | 版本 | 选型理由 |
|------|------|---------|
| Spring Boot | 2.7.x | 快速开发、生态完善、易于部署 |
| MyBatis-Plus | 3.5.x | 增强MyBatis、减少样板代码、强大的CRUD |
| MySQL | 8.0 | 稳定可靠、支持JSON、性能优秀 |
| Redis | 6.x | 高性能缓存、支持多种数据结构 |
| Vue 3 | 3.3.x | 响应式系统优化、组合式API、性能提升 |
| Element Plus | 2.x | 完善的组件库、中文文档、易用性强 |

## 三、数据库详细设计

### 3.1 用户权限模块表

#### 3.1.1 用户表 (sys_user)
```sql
CREATE TABLE sys_user (
    id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '用户ID',
    username VARCHAR(50) UNIQUE NOT NULL COMMENT '用户名',
    password VARCHAR(100) NOT NULL COMMENT '密码(加密)',
    real_name VARCHAR(50) COMMENT '真实姓名',
    phone VARCHAR(20) UNIQUE COMMENT '手机号',
    email VARCHAR(100) COMMENT '邮箱',
    avatar VARCHAR(255) COMMENT '头像URL',
    user_type TINYINT DEFAULT 1 COMMENT '用户类型: 1-种植户 2-采购商 3-管理员',
    status TINYINT DEFAULT 1 COMMENT '状态: 0-禁用 1-正常',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    INDEX idx_username (username),
    INDEX idx_phone (phone),

## 八、技术难点与解决方案

### 8.1 复杂权限管理

**难点**：多角色、动态权限、细粒度控制

**解决方案**：
1. RBAC模型设计(用户-角色-权限)
2. 使用拦截器+注解实现权限校验
3. 前端动态路由生成
4. Redis缓存权限信息提升性能

### 8.2 二维码溯源

**难点**：溯源码唯一性、信息完整性、查询性能

**解决方案**：
1. 使用雪花算法生成唯一溯源码
2. 设计溯源节点链路记录
3. 使用Redis缓存热门溯源信息
4. 二维码生成使用QRCode库

### 8.3 数据分析性能

**难点**：大数据量统计查询、实时性要求

**解决方案**：
1. 使用索引优化查询
2. 定时任务预计算统计数据
3. Redis缓存统计结果
4. 分页查询避免一次性加载大量数据
5. 前端使用ECharts懒加载

### 8.4 文件上传管理

**难点**：文件大小限制、存储方式、访问速度

**解决方案**：
1. 前端文件大小校验
2. 后端分片上传支持(大文件)
3. 使用本地存储/OSS存储
4. 图片压缩优化
5. CDN加速访问

### 8.5 系统并发性能

**难点**：高并发访问、数据库压力、响应速度

**解决方案**：
1. Redis缓存热点数据
2. 数据库连接池配置优化
3. 接口限流防止恶意请求
4. 异步处理非关键业务
5. 前端防抖节流优化

```

- JDK: 11+
- MySQL: 8.0
- Redis: 6.0
- Nginx: 1.20+

### 10.2 后端部署
```bash
1. 打包
mvn clean package -Dmaven.test.skip=true

2. 上传jar包到服务器

3. 启动应用
nohup java -jar herb-platform.jar --spring.profiles.active=prod > app.log 2>&1 &

4. 配置开机自启(systemd)
```

### 10.3 前端部署
```bash
1. 打包
npm run build

2. 上传dist目录到服务器

3. Nginx配置
server {
    listen 80;
    server_name your-domain.com;

    location / {
        root /var/www/herb-platform;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    location /api {
        proxy_pass http://localhost:8080;
    }
}

4. 启动Nginx
nginx -s reload
```

### 10.4 数据库部署

1. 安装MySQL
