## 立即可执行的操作

### 步骤1: 初始化数据库（5分钟）

```bash
# 方式一：命令行
mysql -u root -p
CREATE DATABASE herb_platform DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
USE herb_platform;
source database/schema.sql
source database/data.sql

# 方式二：使用数据库工具
# 1. 打开Navicat/DBeaver等工具
# 2. 新建连接，连接到MySQL
# 3. 新建数据库 herb_platform
# 4. 运行 schema.sql
# 5. 运行 data.sql
```

**验证**：
```sql
USE herb_platform;
SHOW TABLES;  -- 应该看到20张表
SELECT * FROM sys_user;  -- 应该看到3个用户
```

### 步骤2: 配置后端连接信息（2分钟）

编辑 `backend/src/main/resources/application-dev.yml`:

```yaml
spring:
  datasource:
    username: root          # 改成你的MySQL用户名
    password: your_password # 改成你的MySQL密码

  redis:
    password:  # 如果Redis有密码，填写在这里
```

   位置: backend/src/main/java/com/herb/platform/common/Constants.java

   位置: backend/src/main/java/com/herb/platform/exception/BusinessException.java

#### 3.5 生成用户认证模块（完整）

**提示词**：
```
请为用户认证模块生成完整的代码，包括：

【实体类】
User - 用户实体
对应表: sys_user
字段参考: database/schema.sql中的sys_user表
使用: @TableName, @TableId, @TableField, @TableLogic
继承: BaseEntity（需要先创建BaseEntity基类）

【DTO】
1. UserLoginDTO - 登录请求
   - username (用户名)
   - password (密码)
   校验: @NotBlank

2. UserRegisterDTO - 注册请求
   - username
   - password
   - realName
   - phone
   - email
   - userType
   校验: @NotBlank, @Email, @Pattern

3. UserUpdateDTO - 更新用户信息
   - realName
   - phone
   - email
   - avatar

【VO】
1. UserVO - 用户信息展示
   包含: id, username, realName, phone, email, avatar, userType, status, createTime
   不包含: password, deleted

2. UserLoginVO - 登录返回
   - token
   - userInfo (UserVO)
   - permissions (List<String>)

【Mapper】
UserMapper接口
继承: BaseMapper<User>
自定义方法:
   - User selectByUsername(String username)
   - User selectByPhone(String phone)

【Service】
AuthService接口和实现类
方法:
   - UserLoginVO login(UserLoginDTO dto)
   - void logout(String token)
   - UserLoginVO register(UserRegisterDTO dto)
   - UserVO getCurrentUser()

UserService接口和实现类
方法:
   - Page<UserVO> getUserList(QueryRequest request)
   - UserVO getUserById(Long id)
   - void updateUser(Long id, UserUpdateDTO dto)
   - void updatePassword(Long id, String oldPassword, String newPassword)
   - void updateStatus(Long id, Integer status)
   - void deleteUser(Long id)

【Controller】
1. AuthController
   POST /auth/login - 登录
   POST /auth/logout - 登出
   POST /auth/register - 注册
   GET /auth/info - 获取当前用户信息

2. UserController
   GET /users - 分页查询用户列表
   GET /users/{id} - 获取用户详情
   PUT /users/{id} - 更新用户
   PUT /users/{id}/password - 修改密码
   PUT /users/{id}/status - 更新状态
   DELETE /users/{id} - 删除用户

要求:
- 完整的Swagger注解
- 参数校验
- 统一返回Result
- 异常处理
- 日志记录

保存到对应目录:
- entity/
- dto/
- vo/
- mapper/
- service/ 和 service/impl/
- controller/
```

## 常见问题快速解决

### 问题1: Maven依赖下载慢
```xml
<!-- 在pom.xml中添加国内镜像 -->
<repositories>
    <repository>
        <id>aliyun</id>
        <url>https://maven.aliyun.com/repository/public</url>
    </repository>
</repositories>
```

### 问题2: 端口被占用
```bash
# Windows
netstat -ano | findstr :8080
taskkill /PID <PID> /F

# macOS/Linux
lsof -i :8080
kill -9 <PID>
```

### 问题3: 数据库连接失败
```
1. 检查MySQL是否启动
2. 检查用户名密码是否正确
3. 检查数据库是否存在
4. 检查防火墙设置
```

### 问题4: Redis连接失败
```
1. 检查Redis是否启动: redis-cli ping
2. 检查配置文件中的host、port
3. 检查密码配置
```
